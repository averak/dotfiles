# Global Conventions

## 自然言語の書き方

- 対象: code comment / docstring / docs files / PR description など、コード以外のすべての自然言語。
- 1文の途中で改行を入れないこと。複数文あって読みやすくなる場合に限り、1文1行に分けること。
- code comment には why を書くこと。非自明な契約 (どんな入力でエラーになるか、空値を渡すと何が起きるか、など)、採用しなかった代替案、不変条件・順序・並行性・落とし穴、読者が「なぜこう書くのか」と問う箇所の意図を残す。
- code comment に describe-only を書かないこと。識別子・シグネチャ・実装をなぞるだけのコメントや、コード変更で陳腐化する腐るリスト (subcommand 一覧、フィールド名一覧など) は不要。
- コメントで補わず、型で表現できるものは型で表現すること。型・単位・制約をコメントで注記するより、それを型で表現する。コメントは腐るが型は腐らない。
- docstring / doc comment は sparse に保ち、本当に有益なものだけ残すこと。手続き的な why は doc comment ではなく実装箇所の code comment に書く。
